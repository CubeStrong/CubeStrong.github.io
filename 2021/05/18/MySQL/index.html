MySQL学习

DDL（Data Definition Language）数据定义语句-操作数据库和表

基本语法：

三个关键词： create alter drop
create database 数据库名; – 创建数据库
show databases; – 显示所有数据库
use 数据库名; – 使用数据库
show tables; – 显示某个数据库中的所有表
drop database 数据库名; – 删除某个数据库
– 删除数据库会将下面的所有表一起删除
desc 表名 – 查看表
– 创建表
create table 表名 (
列名1 数据类型1 [约束]，
列名2 数据类型2 [约束]
)
– 约束条件有多个用空格隔开

drop table 表名 – 删除表

– 修改表
修改表名：alter table 表名 rename to 新表名
修改表的字符集：alter table 表名 character set 字符集名称（如utf-8）
添加一列：alter table 表名 add 列名 数据类型 [约束]
修改列名称: alter table 表名 change 列名 新列名 数据类型
修改列的数据类型: alter table 表名 modify 列名 数据类型
删除列:alter table 表名 drop 列名
修改列的添加位置：alter table 表名 add 列名1 after 列名2 (将新增的列名1放在列名2的后面)
将某列放在最前面：alter table 表名 modify 列名 数据类型 first

DML（Data Manipulation Language）数据操纵语句-操作数据

关键字：update,delete, insert

DQL（Data Query Language）-数据查询

关键字：select

分页查询：select 字段列表 from 表名 limit 开始的索引,每页查询的条数

​ 开始的索引=（当前页数-1）*每页显示的条数

​ 如：一页显示三条，有三页

​ limit 0,3 –第一页

​ limit 3,3 – 第二页

​ limit 6,3 – 第三页

子查询：select 字段列表 from 表名 where 字段 in (select … ), 和其配套的有not in意为不在某个集合中，使用子查询返回的是一列数据，对于复杂的可以子查询再嵌套子查询。

比较子查询：使表达式的值与子查询的结果进行比较运算

格式：表达式 + <, >, >=, <=, <>, !=, =, !<,!>+ALL,SOME,ANY

ALL指定表达式要与子查询结果集中每一个值都要比较，且满足比较的关系才返回TRUE

SOME和ANY表示表达式只要与子查询结果集中的某个值满足比较关系时就返回TRUE

如：查找比所有计算机系学生年龄都大的学生

多表查询，表之间必须有可比字段

select 字段列表 from 表列表 where 表之间连接的条件以及筛选条件

如查询选修“计算机基础”课程且成绩在80分以上的学生学号、姓名、课程名、成绩

以join关键字指定的连接

语法：select 字段列表 from 左表源 join 右表源 on 连接条件 where 条件，连接分为内连接(inner join)，完全外连接(full outer join)，左外连接(left outer join)，右外连接(right outer join)，交叉连接(cross join)-将两表笛卡尔积运算，其中内连接是系统默认的，可以省略inner关键字,outer 关键字也可以省略。

例如：查询选修206号课程且成绩在80分以上的学生姓名和成绩

查询选修了”计算机基础”课程且成绩在80分以上的学生学号、姓名、课程名、成绩

查询所有学生情况以及他们选修的课程号，若学生未选修任何课，也要包括其情况

查询被选修了的课程的选修情况和所有开设的课程名

列出学生所有可能的选课情况

指定查询结果分组条件，group by

求各专业的学生数

指定查询结果分组统计条件：group by having 条件,对分组之后的结果再筛选，having之后可跟聚合函数，where之后不能跟聚合函数

指定查询结果排序顺序：order by,默认为ASC（升序），DESC（降序）
